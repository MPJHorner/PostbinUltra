Postbin Ultra captures every HTTP request that hits its capture port (any method, any path), prints a colour-coded line in your terminal, and shows the full detail in a live web UI. Optional --forward turns it into a transparent proxy that relays each request to an upstream URL and returns the upstream response. Optional --log-file writes captured requests as NDJSON to a file you can tail or feed into other tools (handy when pairing with an AI assistant that should see incoming traffic).

Usage: postbin-ultra [OPTIONS]

Options:
  -p, --port <PORT>
          Port the capture server listens on. Send your requests here
          
          [default: 9000]

  -u, --ui-port <UI_PORT>
          Port the web UI listens on
          
          [default: 9001]

      --bind <BIND>
          Bind address for both servers
          
          [default: 127.0.0.1]

      --max-body-size <MAX_BODY_SIZE>
          Maximum captured body size in bytes. Larger bodies are truncated
          
          [default: 10485760]

      --buffer-size <BUFFER_SIZE>
          Number of requests to keep in memory
          
          [default: 1000]

      --no-ui
          Disable the web UI server entirely

      --no-cli
          Disable the colored CLI output

      --json
          Emit each request as JSON (NDJSON) to stdout

      --open
          Open the web UI in your browser on startup

  -v, --verbose
          Verbose CLI output: prints headers and a body preview for each request

      --update
          Download the latest release from GitHub and replace this binary, then exit

      --no-update-check
          Skip the startup check that asks GitHub if a newer release is available

      --forward <URL>
          Forward each captured request to URL and return the upstream response to the original caller. Path and query are appended to URL

      --forward-timeout <FORWARD_TIMEOUT>
          Per-request timeout in seconds when forwarding upstream
          
          [default: 30]

      --forward-insecure
          Skip TLS certificate verification when forwarding (dev/staging only)

      --log-file <FILE>
          Append every captured request to FILE as one JSON object per line (NDJSON). The file is created if missing and never truncated. Useful when pairing with --forward to give an AI assistant or other tool a tail-able feed of requests as they arrive

  -h, --help
          Print help (see a summary with '-h')

  -V, --version
          Print version
